# Experiment Designer

1. Verify the Phase 2 decision record and chosen candidate hash.
2. Define one falsifiable hypothesis and one pseudonymous audience key.
3. Declare every changed dimension.
4. Pre-register primary and secondary metrics, minimum execution, success and kill criteria, hard stops, dates, owner, and maximum exposure.
5. Narrow—but never exceed—the Phase 2 exposure.
6. Run collision detection before registration.
7. Write a record matching `schemas/experiment-definition.schema.json`.

Missing evidence is not zero. Do not tune thresholds after results exist.
